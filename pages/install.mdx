# Install

The Omen CLI ships via a single-line installer that pulls the latest
release from our CDN and installs it globally via `npm`.

## Requirements

- **Node.js 22 or newer** (`node -v` to check).
  If you don't have it, grab it from [nodejs.org](https://nodejs.org) —
  or via [nvm](https://github.com/nvm-sh/nvm), [fnm](https://github.com/Schniz/fnm),
  [volta](https://volta.sh), or [asdf](https://asdf-vm.com).
- **npm** (ships with Node).
- A terminal. An interactive TTY is required to run the TUI.

## macOS and Linux

```bash
curl -fsSL https://install.usefirmware.com/omen/install.sh | sh
```

## Windows (PowerShell)

```powershell
iwr -useb https://install.usefirmware.com/omen/install.ps1 | iex
```

## First run

```bash
omen login         # opens your browser, authorizes the device
omen               # launches the interactive assistant
```

See [CLI commands](/commands) for the full reference.

## Pin a specific version

Useful for CI or when you want to reproduce an exact build.

```bash
# macOS / Linux
curl -fsSL https://install.usefirmware.com/omen/install.sh | OMEN_VERSION=0.2.0 sh
```

```powershell
# Windows
$env:OMEN_VERSION = "0.2.0"; iwr -useb https://install.usefirmware.com/omen/install.ps1 | iex
```

Old versions stay available on the CDN, so pinning doesn't expire.

## Headless / no browser

If you're on a remote box without a browser (SSH, CI), use the paste flow —
the web app prints the token and you paste it into the CLI.

```bash
omen login --paste
```

## Update

```bash
omen update
```

Re-runs the installer to pull the newest release. Same path for every
platform, regardless of which package manager you installed under.

## Uninstall

```bash
omen logout              # forget the token + local project state
npm uninstall -g omen-cli
```

## Troubleshooting

### `omen: command not found`

npm's global bin directory isn't on your `PATH`. Find it with:

```bash
npm prefix -g
```

Add `<that path>/bin` to your shell's `PATH` (typically in `~/.zshrc`
or `~/.bashrc`), then open a new terminal.

### `EACCES` during install on macOS / Linux

You're installing globally under a system-owned Node. Fix the permission
once — don't keep using `sudo`. Follow npm's guide:
[Resolving EACCES errors](https://docs.npmjs.com/resolving-eacces-permissions-errors).

Alternatively, use a version manager (`nvm` / `fnm` / `volta`) which
installs Node under your home directory and sidesteps the issue.

### "Node.js 22+ required, found v18.x"

Upgrade Node — the CLI uses features that landed in Node 22.

### Install hangs on Windows

Run PowerShell as Administrator. npm's global install on Windows
sometimes needs elevated perms depending on where Node was installed.
